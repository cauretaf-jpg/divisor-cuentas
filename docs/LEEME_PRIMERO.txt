IMPORTANTE SUPABASE:
Para usar cuentas compartidas debes ejecutar este archivo nuevo en Supabase:
1. sql/03-supabase-shared-accounts.sql

Si aún no lo hiciste antes, también ejecuta:
1. sql/01-supabase-app-state.sql
2. sql/02-supabase-social.sql
3. sql/03-supabase-shared-accounts.sql

Instalación rápida:
1. Cierra el servidor local si está abierto.
2. Reemplaza los archivos de tu carpeta Cuentas por los de este ZIP.
3. Abre index.html con Live Server o usa tu flujo habitual.
4. Recarga el navegador con Ctrl + F5 para limpiar caché.

Nota:
Las cuentas recurrentes funcionan en local/nube personal con app_states. Las cuentas compartidas requieren iniciar sesión y ejecutar sql/03-supabase-shared-accounts.sql.


V9.2: si aparece "new row violates row-level security policy for table shared_accounts", ejecutar nuevamente sql/03-supabase-shared-accounts.sql. Esta versión usa funciones seguras create_shared_account_safe/update_shared_account_safe para crear y actualizar cuentas compartidas con auth.uid().
